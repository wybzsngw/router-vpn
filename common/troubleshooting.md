# Clash 常见问题排查指南

本文汇总 Clash 在各平台上的常见问题及排查思路，帮助你快速定位并解决问题。

---

## 一、订阅相关

### 1.1 订阅导入失败

**现象**：粘贴订阅链接后提示失败或无法获取配置。

**可能原因与处理**：

- **网络问题**：检查本机能否访问订阅域名，可尝试用浏览器直接打开订阅链接测试。
- **链接格式错误**：确认复制完整，无多余空格或换行。
- **订阅链接过期**：登录机场控制台检查订阅是否有效，必要时重新生成。
- **客户端限制**：部分客户端对订阅链接格式有要求，可尝试使用订阅转换服务生成 Clash 配置。

### 1.2 节点列表为空

**现象**：订阅更新成功，但节点列表为空。

**可能原因与处理**：

- **订阅链接已失效**：重新生成订阅链接。
- **账户欠费或套餐过期**：确认账户状态并续费。
- **订阅链接类型不匹配**：确认使用的是 Clash 订阅链接，而非 Base64 节点列表等。
- **订阅转换服务异常**：若使用转换服务，检查转换服务是否正常。

### 1.3 订阅过期

**现象**：提示订阅已过期或无法更新。

**可能原因与处理**：

- **套餐到期**：登录机场控制台续费。
- **订阅链接有时效**：部分机场的订阅链接带有时效参数，重新生成即可。

### 1.4 订阅格式错误

**现象**：提示配置格式错误或解析失败。

**可能原因与处理**：

- **非 Clash 格式**：使用订阅转换服务生成 Clash 配置。
- **YAML 语法错误**：若为手动编辑，检查缩进、冒号、引号等是否正确。

## 二、连接相关

### 2.1 无法连接

**现象**：选择节点后无法连接或一直转圈。

**可能原因与处理**：

- **节点失效**：更换其他节点尝试。
- **本地网络限制**：检查路由器、防火墙、公司网络是否限制代理端口。
- **协议/端口被封锁**：尝试更换协议或使用带 TLS 的节点（如 Trojan）。
- **代理模式未开启**：确认系统代理或 TUN 模式已正确启用。

### 2.2 速度慢

**现象**：连接成功但速度很慢。

**可能原因与处理**：

- **节点负载高**：更换节点或使用负载较低的节点。
- **线路质量差**：选择 IPLC、IEPL、专线等优质线路。
- **DNS 解析慢**：检查 DNS 配置，可尝试 `fake-ip` 或更换 DNS 服务器。
- **本地带宽**：确认本机带宽和网络环境正常。

### 2.3 频繁断线

**现象**：连接后频繁断开或重连。

**可能原因与处理**：

- **节点不稳定**：更换节点或使用 `url-test` 策略组自动选择。
- **网络波动**：检查本地网络稳定性。
- **客户端问题**：更新 Clash 版本或尝试其他客户端。

### 2.4 特定网站无法访问

**现象**：部分网站打不开或仍被封锁。

**可能原因与处理**：

- **分流规则未生效**：检查规则是否命中，可临时使用全局代理测试。
- **DNS 污染**：启用 `fake-ip` 或更换 DNS。
- **网站封禁代理 IP**：更换节点或使用住宅 IP 节点。

## 三、DNS 相关

### 3.1 DNS 泄露

**现象**：DNS 请求未经过代理，暴露真实 IP。

**可能原因与处理**：

- **系统 DNS 未接管**：使用 TUN 模式时，确保 Clash 接管系统 DNS。
- **规则未覆盖**：某些 DNS 请求可能绕过代理，检查规则和 DNS 配置。
- **IPv6 泄露**：检查是否启用 IPv6，必要时关闭或单独配置。

### 3.2 DNS 解析失败

**现象**：无法解析域名或解析超时。

**可能原因与处理**：

- **DNS 服务器不可达**：更换 DNS（如 `8.8.8.8`、`1.1.1.1`）。
- **DNS 被污染**：使用 `fake-ip` 或 DoH/DoT。
- **规则冲突**：检查 DNS 规则是否将某些域名错误直连。

### 3.3 DNS 解析慢

**现象**：打开网页慢，首屏加载明显延迟。

**可能原因与处理**：

- **DNS 服务器延迟高**：更换更快的 DNS。
- **fake-ip 与规则配合**：某些场景下 `redir-host` 可能更合适，可尝试切换。

## 四、端口相关

### 4.1 端口冲突

**现象**：Clash 启动失败，提示端口被占用。

**可能原因与处理**：

- **其他程序占用**：修改 Clash 的 HTTP/ SOCKS5/混合端口，或关闭占用端口的程序。
- **多实例冲突**：确保只运行一个 Clash 实例。

### 4.2 端口被占用

**现象**：无法绑定指定端口。

**可能原因与处理**：

- **权限不足**：Linux 下 1024 以下端口需 root 权限。
- **防火墙拦截**：检查防火墙规则，放行对应端口。

### 4.3 防火墙拦截

**现象**：本机可访问，其他设备无法通过 Clash 代理。

**可能原因与处理**：

- **允许局域网连接**：在 Clash 配置中开启 `allow-lan: true`。
- **防火墙放行**：在系统防火墙中放行 Clash 的 HTTP/SOCKS5 端口。

## 五、TUN 模式相关

### 5.1 TUN 模式不生效

**现象**：开启 TUN 后流量仍走系统代理或直连。

**可能原因与处理**：

- **驱动未安装**：Clash 需要 TUN 驱动，确认已正确安装。
- **权限不足**：Windows 需管理员权限，Linux 需 root。
- **路由表未生效**：检查系统路由表，确认 TUN 设备路由正确。

### 5.2 需要管理员/root 权限

**现象**：提示需要管理员或 root 权限。

**可能原因与处理**：

- **TUN 模式**：创建虚拟网卡需要高权限。
- **解决方案**：以管理员身份运行（Windows）或使用 `sudo`（Linux）。

### 5.3 驱动问题

**现象**：TUN 驱动安装失败或无法加载。

**可能原因与处理**：

- **Windows**：安装 Wintun 或 tap-windows 驱动。
- **macOS**：检查 utun 或相关驱动是否正常。
- **Linux**：确认内核支持 TUN，`modprobe tun` 加载模块。

## 六、流媒体相关

### 6.1 Netflix/Disney+ 仍无法访问

**现象**：使用代理后仍提示地区限制或无法播放。

**可能原因与处理**：

- **节点未解锁**：选择支持流媒体解锁的节点。
- **IP 被识别为代理**：部分流媒体封禁数据中心 IP，需住宅 IP 或特殊解锁节点。
- **Cookie 缓存**：清除浏览器缓存和 Cookie 后重试。

### 6.2 地区错误

**现象**：流媒体显示的地区与预期不符。

**可能原因与处理**：

- **节点落地 IP 地区**：确认节点所在地区与流媒体要求一致。
- **分流规则**：确保流媒体域名走专用节点组。

### 6.3 IP 被封禁

**现象**：提示「您使用的网络可能使用了代理」等。

**可能原因与处理**：

- **更换节点**：使用未被封禁的节点。
- **使用住宅 IP**：部分机场提供住宅 IP 节点，可尝试。

## 七、性能相关

### 7.1 高 CPU 占用

**现象**：Clash 进程 CPU 占用高。

**可能原因与处理**：

- **规则过多**：精简规则集，减少规则数量。
- **频繁测速**：降低 `url-test` 间隔。
- **大量连接**：检查是否有异常连接或恶意流量。

### 7.2 内存占用高

**现象**：Clash 内存持续增长。

**可能原因与处理**：

- **规则集过大**：减少规则集或使用更轻量的规则。
- **客户端 Bug**：更新到最新版本，或尝试其他客户端。

### 7.3 启动慢

**现象**：Clash 启动或切换配置耗时较长。

**可能原因与处理**：

- **规则集远程加载**：首次加载需下载，可预缓存或使用本地规则。
- **节点数量多**：减少节点数量或使用策略组筛选。
