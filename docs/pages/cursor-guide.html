最重要的功能

Agent 模式：跟 AI 聊天，就能搞定网站

这是 Cursor 最强大、也最适合普通人的功能。你说需求，AI 自己规划步骤、写代码、修 bug、直到完成任务。

传统的 AI 聊天（比如 ChatGPT）是一问一答——你问一个问题，它给你一段回答，你还得自己把代码复制粘贴进去，还要自己搞清楚哪里出了问题。

Cursor Agent 不一样：它可以直接读你电脑上的文件、写进去、改掉、还能自己运行命令验证结果——你只需要看它干完后说"OK"或"这里不对，改一下"。

怎么用 Agent？

  1. 按 Ctrl+Shift+I（macOS：Cmd+Shift+I）打开 Composer
  2. 在输入框里用中文描述你想做的事（任务越具体越好）
  3. Agent 会列出计划，开始逐步执行，你在旁边看着
  4. 如果有步骤不对，直接在输入框里说"这里不对，应该..."
  5. 完成后检查结果，觉得 OK 就接受，不对就让它改

「帮我建一个个人主页，简洁风格，有自我介绍和联系方式」生成 HTML/CSS 文件，自动格式化，可直接上传 GitHub

💡 提示词技巧：任务描述越具体，结果越好。告诉 Agent：你想要什么（目标）、有什么限制（约束）、怎么判断做好了（验收标准）。三段式结构效果最佳。
其他常用功能

Chat / Tab / Cmd+K — 三个轻量助手

功能快捷键最适合的场景
Tab 补全 写代码时按 Tab AI 根据上下文自动预测你下一行要写什么，按 Tab 接受。不想要就继续打字忽略。
Cmd+K 快改 选中代码 → Ctrl/Cmd+K 选中一段代码，告诉 AI 你想怎么改（如"把这个函数改成用英文注释"），它直接在原地改好。
Chat 问答 右侧 Chat 面板 问问题、让 AI 解释代码、分析问题、给建议。不会直接改文件，只给你回答。

@ 引用 — 让 AI 看你的文件

在 Chat 或 Composer 里，输入 @ 可以引用：

  • @文件名 — 让 AI 读取某个文件的内容再回答
  • @文件夹 — 让 AI 了解整个目录结构
  • @代码片段 — 引用当前光标位置的函数或类

例如：@index.html 帮我找一下这个文件里有没有写重复的 CSS 类

遇到问题

常见故障排查

登录不进去 / 反复要求验证

清除 Cursor 缓存（Help → Reset App Data），用无痕窗口重新登录；确认代理软件没有拦截 Cursor 的请求。

模型不响应 / 一直转圈

先检查网络：切换到 Composer-2（免费模型）试试。如果 Composer-2 正常，说明海外模型（Claude/GPT）需要稳定海外网络。切换节点后再试。

用了很多次后提示「额度用完」

Free 档有月度用量上限。Pro 档对 Sonnet 4.6 有大量配额，偶尔切 Composer-2 可以省下高端模型额度用于关键任务。

付款后权益没更新

先刷新页面、重新登录。等 5 分钟仍未更新，进入 Settings → Account 查看订阅状态，或通过 Cursor 官方支持渠道提交订单号。

GitHub 推送时提示认证失败

在 Cursor 里重新授权 GitHub：Settings → 搜索「GitHub」→ 重新登录授权，再尝试推送。
