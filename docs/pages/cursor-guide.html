① Cursor 是什么

Cursor 是把 AI 深度融合进工程工作流的编程环境。它通过 Tab、Cmd+K、Chat、Composer/Agent 四类入口，把需求、实现、验证串成一个更短闭环。

官方模型文档将个人用量分为 Auto + Composer 池与 API 池，这也是我们建议按任务阶段选模型的重要依据。

② 下载与注册

  1. 安装对应系统版本并完成登录；
  2. 进入设置页确认模型与套餐可见性；
  3. 用小项目测试 Tab、Cmd+K、Chat 三种基础流程。

建议先固定提示词模板和验证清单，后续效率会更稳定。

⑥ 模型选型

Cursor 官方将 Auto 定义为在智能、成本与可靠性间平衡的策略入口；Composer 2 是针对 agentic coding 强化训练的自研模型。

  1. 需求澄清阶段：Auto / Composer 2；
  2. 关键实现阶段：Claude 4.5 / GPT-5；
  3. 回归修补阶段：切回 Auto 或 Composer 2。
