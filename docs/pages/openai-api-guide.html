Section 02

申请 API 前需要准备什么

  1. 一个正常可用的 OpenAI 账号：能登录 ChatGPT 最好。
  2. 稳定的网络环境：你需要能访问 platform.openai.com，优先使用美国、新加坡或服务商标注的 AI 解锁节点。
  3. 支付方式：很多账户在实际调用前需要先给 API 钱包充值。
  4. 基础安全意识：API Key 一旦泄露，别人就能直接消费你的余额。
建议：如果你连 ChatGPT 网页端都还没稳定跑通，先回去看 ChatGPT 主教程，把网络和账号先搞定。
Section 03

如何获取 OpenAI API Key

  1. 登录 platform.openai.com。
  2. 进入开发者平台，查看项目、用量、账单等页面。
  3. 在密钥管理页面创建新的 API Key。
  4. 复制后立刻保存到安全位置，例如本地密码管理器或服务器环境变量。
  5. 不要把 Key 写进公开仓库、前端源码或截图里。
安全提醒：API Key 相当于你的钱包权限。任何泄露到 GitHub、前端 JS、公开日志里的 Key，都应该立刻作废并重建。

在代码里推荐通过环境变量使用，例如读取 OPENAI_API_KEY，而不是硬编码在文件里。

Section 05

适合哪些人用 API

  • 开发者：需要把 AI 集成进自己的网站、应用、机器人或脚本。
  • 自动化用户：想批量处理文本、表格、分类、摘要、翻译。
  • 产品运营：要做表单自动回复、客服草稿、内容生成流程。
  • 非开发用户：如果只是自己聊天、写文案、问问题，优先用 ChatGPT 网页版，不一定需要 API。
Section 06

常见问题 FAQ

Q1：开了 ChatGPT Plus，还要不要单独充值 API？

A：要。Plus 和 API 是两套独立计费系统，Plus 不会自动附带 API 额度。

Q2：API Key 创建后为什么只显示一次？

A：这是正常的安全设计。很多平台只在创建当下展示完整密钥，之后不会再重复显示，所以你要第一时间保存好。

Q3：API 能不能放到前端网页里直接调用？

A：不建议。前端代码会暴露给用户，API Key 很容易泄露。正确做法是把调用放在你自己的后端或服务器里。

Q4：中国用户用 API 最容易卡在哪？

A：通常是网络环境、平台访问限制，以及对 Plus 和 API 计费关系的误解。很多人以为已经开了 Plus 就不用再管 API 账单，这一步最容易踩坑。关于代理对 API 访问究竟能做什么、不能做什么（包括 IP 类型与机房 IP 的区别），可参考：代理能做什么、不能做什么：AI 访问与住宅 IP 真相。

Q5：没有国际信用卡，能不能把账号交给第三方处理付款？

A：不建议。API Key 一旦泄露可能被刷量，造成真实账单损失。API 充值建议优先自己操作；如果只是想用 GPT-5.5 / Codex 完成文档修改、代码开发、故障排查，可以考虑通过 Cursor IDE 使用，不要把 Key 交给第三方。
