## 一、软件介绍

### 什么是 Clash Meta for Android？

**Clash Meta for Android**（原名 ClashMetaForAndroid，简称 CMA）是一款基于 Clash Meta 内核的 Android 代理客户端。它支持订阅管理、规则分流、TUN 模式、分应用代理等功能，是 Android 平台上最流行的 Clash 客户端之一。

### 为什么选择 Clash Meta for Android？

- **开源免费**：完全开源，代码透明可审计
- **功能完善**：支持订阅、规则、TUN、分应用代理等
- **持续维护**：社区活跃，定期更新
- **兼容性强**：支持 Clash、Clash Meta 等主流订阅格式

### 官方发布页

- **GitHub Releases**：<https://github.com/MetaCubeX/ClashMetaForAndroid/releases>
- 建议从官方 GitHub 或 F-Droid 下载，确保安全可靠

## 二、系统要求

| 项目 | 要求 |
|------|------|
| **操作系统** | Android 7.0 (API 24) 或更高 |
| **架构** | ARM64 推荐，兼容 ARMv7、x86、x86_64 |
| **磁盘空间** | 约 50MB |
| **权限** | 需授予 VPN 权限（系统级要求） |

### 建议

- **ARM64** 设备（绝大多数 2016 年后发布的手机）性能最佳
- 部分老旧设备可能仅支持 ARMv7，需下载对应架构的 APK

## 三、下载安装

### 1. 从 GitHub 下载

1. 在手机浏览器中打开：<https://github.com/MetaCubeX/ClashMetaForAndroid/releases>
2. 找到最新版本（Latest），点击展开 **Assets**
3. 根据您的设备架构选择：
   - **arm64-v8a** — 绝大多数 64 位 Android 手机（推荐）
   - **armeabi-v7a** — 32 位 ARM 设备
   - **x86_64** — 模拟器或 x86 设备
4. 点击对应的 `.apk` 文件下载

### 2. 从 F-Droid 下载（可选）

若已安装 F-Droid 应用商店，可在 F-Droid 中搜索 **Clash Meta for Android** 进行安装，便于后续自动更新。

### 3. 不在 Google Play 上架

Clash Meta for Android **未在 Google Play 上架**，因此无法从国内或海外 Google Play 直接下载。请务必从 GitHub 或 F-Droid 获取。

### 4. 允许安装未知来源应用

若系统提示「无法安装未知来源的应用」，请按以下步骤操作：

1. 打开 **设置** → **应用** → **特殊应用权限**（或 **安全与隐私**）
2. 找到 **安装未知应用** 或 **未知来源**
3. 选择您用于下载的浏览器（如 Chrome、系统浏览器）
4. 允许该应用安装未知应用

### 5. 安装 APK

1. 打开文件管理器，找到下载的 `.apk` 文件
2. 点击安装，按提示完成安装
3. 安装完成后，在桌面或应用列表中打开应用

## 四、导入订阅

### 1. 获取订阅链接

首先需要从代理服务商获取订阅链接（Subscription URL），通常为一串以 `https://` 开头的 URL。若您已有路由器 Clash 方案，可复用同一订阅链接。

### 2. 添加订阅步骤

1. 打开 **Clash Meta for Android**
2. 点击底部 **Profile**（配置）标签
3. 点击右上角 **+** 按钮
4. 选择 **URL** 方式
5. 在 **URL** 输入框中粘贴您的订阅链接
6. **Profile Name** 可自定义（如「我的订阅」）
7. 点击 **Save**（保存）或 **确定**

### 3. 更新订阅

添加后，应用会自动拉取订阅内容。若需手动更新：

- 在 Profile 列表中，长按该配置 → 选择 **Update**（更新）
- 或点击配置右侧的刷新图标

### 4. 订阅导入失败

若提示「导入失败」或「解析错误」，请检查：

- 订阅链接是否完整、可访问
- 网络是否正常（可先尝试连接 Wi-Fi 或移动数据）
- 订阅格式是否被 Clash Meta 支持（常见格式如 Clash、Clash Meta 均支持）

## 五、启动连接

### 1. 选择配置

在 **Profile** 中，确保已选中您要使用的订阅配置（通常显示为蓝色勾选）。

### 2. 启动代理

1. 返回应用主界面

2. 点击中央的 **大按钮** 或 **开关** 启动代理

3. 首次启动时，系统会弹出 **VPN 权限请求**：
   - 点击「确定」或「允许」授予 VPN 权限
   - 若拒绝，应用将无法建立代理连接

4. 连接成功后，按钮会变为「已连接」状态，通知栏或状态栏会显示 VPN 图标

### 3. 断开连接

再次点击大按钮或开关即可断开代理。

## 六、代理模式

Clash Meta for Android 支持多种代理模式，可根据需求切换：

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **Rule（规则）** | 按规则分流：国内直连，国外走代理 | 日常使用，推荐 |
| **Global（全局）** | 所有流量走代理 | 需要代理所有应用时 |
| **Direct（直连）** | 所有流量直连，不走代理 | 临时关闭代理时 |

### 如何切换模式

1. 在主界面或 **Proxies** 标签中
2. 找到顶部的模式选择器（通常显示为 Rule / Global / Direct）
3. 点击切换即可

部分订阅可能使用自定义策略组名称，如「自动选择」「节点选择」等，可在 **Proxies** 标签中查看和切换。

## 七、分应用代理

分应用代理可指定哪些应用走代理、哪些直连，节省流量并提升体验。

### 1. 开启分应用代理

1. 打开 **Clash Meta for Android**
2. 点击底部 **Settings**（设置）
3. 进入 **Network**（网络）→ **Per-App Proxy**（分应用代理）
4. 开启该功能

### 2. 白名单模式

- **Bypass（绕过）**：只对列表中的应用走代理，其余直连
- 适用于：仅需代理少数应用（如浏览器、Twitter、YouTube）

### 3. 黑名单模式

- **Proxy（代理）**：只对列表中的应用直连，其余走代理
- 适用于：大部分应用走代理，少数应用（如银行、支付）直连

### 4. 添加应用

1. 在分应用代理设置中，点击「添加应用」或「+」
2. 从应用列表中选择要加入的应用
3. 保存后即可生效

## 八、节点选择与测速

### 1. 进入 Proxy 标签

点击底部 **Proxies**（代理）标签，可看到订阅中的节点列表和策略组。

### 2. 选择节点组

- 若订阅包含「自动选择」「节点选择」等策略组，点击进入
- 在节点列表中点击任意节点即可切换

### 3. 测速

1. 在节点列表或策略组中，点击 **测速** 或 **延迟测试** 按钮
2. 应用会逐个测试节点延迟（ping）
3. 测速完成后，可按延迟排序，选择延迟最低的节点

### 4. 按速度排序

- 测速后，节点通常会自动按延迟从低到高排序
- 可手动选择延迟低、稳定性好的节点使用

---

## 九、自动更新订阅

订阅会定期更新节点列表，建议开启自动更新：

### 1. 设置自动更新

1. 打开 **Settings**（设置）
2. 进入 **Profile Auto Update**（配置自动更新）
3. 开启自动更新
4. 设置更新间隔，如 **24 小时**、**12 小时** 等

### 2. 注意事项

- 更新间隔不宜过短，以免频繁请求订阅链接被服务器限制
- 建议 24 小时更新一次即可
- 若订阅链接失效，需手动更换新链接

## 十、常见问题

### 1. VPN 权限被拒绝

**现象**：点击连接后提示「无法建立 VPN」或「权限被拒绝」。

**解决**：

- 确保在系统弹窗中点击「确定」授予 VPN 权限
- 若已拒绝，可到 **设置** → **应用** → **Clash Meta for Android** → **权限** 中检查权限
- 部分 ROM 需在「VPN 权限」或「VPN 应用」中单独开启

### 2. 后台被系统杀死

**现象**：锁屏或切换应用后，代理自动断开。

**解决**：

- 进入 **设置** → **应用** → **Clash Meta for Android** → **电池**
- 选择「无限制」或「不限制后台」
- 部分厂商需在「自启动管理」中允许应用自启动
- 可开启「后台运行」或「保持后台运行」选项（若有）

### 3. 订阅导入失败

**现象**：添加订阅后提示「获取失败」「解析错误」等。

**解决**：

- 检查订阅链接是否可访问（可在浏览器中打开测试）
- 确认网络正常（可先连接 Wi-Fi 或移动网络）
- 部分订阅可能需在特定网络环境下才能获取（如需先连代理）
- 尝试更换订阅格式（若服务商支持多种格式）

### 4. IPv6 相关问题

**现象**：开启代理后部分网站无法访问，或出现 IPv6 相关错误。

**解决**：

- 在 **Settings** → **Network** 中检查 IPv6 相关设置
- 若订阅或节点不支持 IPv6，可尝试关闭 IPv6 或使用「IPv4 优先」
- 部分运营商 IPv6 环境复杂，可尝试关闭手机 IPv6 测试

### 5. 连接后无法上网

**现象**：点击连接后显示已连接，但无法打开网页。

**解决**：

- 检查订阅是否已正确加载并包含有效节点
- 尝试切换节点或代理模式（Rule / Global）
- 检查规则是否正确（如规则模式下，国内直连可能因规则问题异常）
- 重启应用或手机后重试

## 十一、进阶：与路由器方案互补

若您已在家中部署了路由器 Clash 方案，可结合手机端实现「在家用路由器，出门用手机」的互补使用方式。

### 1. 使用场景

| 场景                         | 推荐方案                   |
| ---------------------------- | -------------------------- |
| 在家（连接家中 Wi-Fi）       | 路由器 Clash，全设备自动代理 |
| 出门（移动网络或公共 Wi-Fi） | 手机 Clash Meta for Android |

### 2. 避免冲突

- **同时开启时**：若手机连接家中 Wi-Fi 且路由器已开启 Clash，手机再开 Clash 可能导致双重代理或冲突
- **建议**：在家时关闭手机端 Clash，让流量走路由器；出门时再开启手机端代理

### 3. 订阅复用

- 手机端和路由器可使用**同一订阅链接**
- 无需单独购买订阅，节省成本
- 节点列表和规则可保持一致

### 4. 配置同步

- 若订阅支持自动更新，手机和路由器都会获取最新节点
- 无需手动同步配置
