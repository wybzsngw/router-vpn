# 软路由翻墙教程 — iKuai + Clash 旁路由

> 📄 本文对应 HTML 页面：[软路由教程](../docs/pages/soft-router-guide.html)

本教程介绍如何以 **iKuai 爱快** 作为主路由，**OpenWrt + OpenClash** 作为旁路由，实现多 WAN、流控、行为管理等功能与翻墙代理的分离部署。

---

## 一、方案概述

### 为什么选择 iKuai + 旁路由？

| 组件 | 职责 | 优势 |
|------|------|------|
| **iKuai 主路由** | 拨号、多 WAN 负载均衡、QoS 流控、行为管理、认证计费 | 企业级功能，适合多线接入、带宽管理 |
| **OpenWrt 旁路由** | 仅负责 Clash 代理 | 专注翻墙，主路由故障不影响代理；可单独重启升级 |

### 职责分离的好处

- **主路由稳定**：iKuai 专注路由与流控，不跑代理，稳定性更高
- **代理独立**：OpenWrt/OpenClash 可随时更新、重启，不影响主网络
- **灵活切换**：需要代理的设备指向旁路由，不需要的走主路由直连

## 三、iKuai 基础配置

### 1. WAN 口配置

- 进入 **网络设置 → 内外网设置**
- 添加 WAN 口，选择 DHCP 或 PPPoE 拨号
- 多 WAN 用户可添加多个 WAN 口并配置负载均衡

### 2. LAN 口配置

- LAN 口 IP：如 `192.168.1.1`，子网掩码 `255.255.255.0`
- 确保与旁路由规划在同一网段

### 3. DHCP 服务器

- 进入 **网络设置 → DHCP 服务端**
- 添加 DHCP 服务：接口选 LAN，地址池如 `192.168.1.100` ~ `192.168.1.200`
- 网关、DNS 可先设为 `192.168.1.1`（主路由）

### 4. 可选：DHCP 推送旁路由为网关

若希望部分或全部客户端自动走代理，可在 DHCP 中配置：

- **方式 A**：创建单独 DHCP 地址池，网关设为 `192.168.1.2`（旁路由），通过 MAC 或 VLAN 分配
- **方式 B**：使用 iKuai 的 DHCP 自定义选项（如 option 3 网关）针对特定客户端推送

## 四、旁路由 OpenWrt 配置

### 1. 网络设置

- **接口**：仅保留 LAN，删除或禁用 WAN
- **LAN 配置**：
  - 协议：静态地址
  - IP：`192.168.1.2`（与 iKuai 同网段）
  - 子网掩码：`255.255.255.0`
  - 网关：`192.168.1.1`（iKuai）
  - DNS：`192.168.1.1` 或 `8.8.8.8`、`1.1.1.1`

### 2. 关闭 DHCP

- 在 LAN 接口设置中，将 DHCP 设为「忽略」或「关闭」
- 避免与 iKuai 的 DHCP 冲突

### 3. 防火墙

- 旁路由通常不需要 NAT，可关闭 LAN 的 IP 动态伪装（MASQUERADE）
- 或保持默认，确保流量能正常转发

### 4. 安装 OpenClash

按照 [OpenWrt + OpenClash 教程](./openwrt-clash.md) 第五、六节完成 OpenClash 安装与订阅导入。

## 五、客户端指向

### 方法 A：单设备手动设置网关

在需要翻墙的设备上：

- **Windows**：网络适配器 → IPv4 → 手动设置网关为 `192.168.1.2`
- **手机/平板**：连接 WiFi 后，在高级设置中修改网关为 `192.168.1.2`
- **智能电视**：若支持静态 IP，网关填 `192.168.1.2`

### 方法 B：iKuai DHCP 推送网关

在 iKuai 的 DHCP 中，为指定客户端或地址池设置网关为 `192.168.1.2`，则这些设备会自动通过旁路由上网。

### 方法 C：iKuai 策略路由

- 使用 iKuai 的 **流控分流 → 分流设置** 或 **端口分流**
- 将特定 IP、MAC 或端口的流量指向旁路由 `192.168.1.2`
- 适合按用户、按应用精细控制

## 六、常见问题

### 1. 路由循环

**现象**：客户端无法上网，或网络极慢。

**原因**：旁路由将流量转发回主路由，主路由又指回旁路由，形成环路。

**解决**：
- 确认旁路由网关指向主路由 `192.168.1.1`，而非自身
- 确认主路由 DHCP 的网关为 `192.168.1.1`（不代理时）或 `192.168.1.2`（代理时），不要出现交叉指向

### 2. DNS 解析循环

**现象**：能 ping 通 IP，但无法打开网页。

**原因**：主路由与旁路由的 DNS 劫持冲突，或 DNS 请求在两者间循环。

**解决**：
- 旁路由 OpenClash 开启 DNS 劫持，客户端 DNS 设为旁路由 `192.168.1.2`
- 主路由关闭对 53 端口的重定向（若存在）
- 或在 OpenClash 中使用 7874 等非 53 端口，避免与 dnsmasq 冲突

### 3. 客户端无法上网

**排查**：
- 客户端网关是否为 `192.168.1.2`？
- 旁路由能否 ping 通主路由 `192.168.1.1`？
- OpenClash 是否已启动？运行模式是否正确？
- 防火墙是否放行转发？

### 4. 速度下降明显

**可能原因**：
- 旁路由性能不足（CPU、内存）
- 节点带宽或延迟不佳
- 未开启硬件 offload / flow offloading

**建议**：检查旁路由负载，更换节点，开启流控加速。
